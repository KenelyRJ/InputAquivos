📁 Apresentação: API de Upload de Arquivos

Foco: Entrada de Arquivo com MultipartFile no Spring Boot

Slide 1: Visão Geral da API

  • API REST para upload e processamento de arquivos (XML/TXT)
  • Endpoint principal: POST /api/v1/arquivos
  • Retorna conteúdo processado + salva em disco
  • GET para visualizar o último arquivo
  • Tecnologia: Spring Boot + MultipartFile

Slide 2: O que é MultipartFile?

MultipartFile é a interface do Spring para lidar com uploads via multipart/form-data.

  • Abstrai o parsing do request HTTP
  • Fornece acesso a: nome, tipo de conteúdo, tamanho, stream de bytes
@PostMapping("/arquivos")
public ArquivoDto processaArquivo(@RequestPart("file") MultipartFile file)
    Obs: Controller é o controlador que recebe as requisição e "invoka" o services para as regras de negócios

Slide 3: Como funciona o Upload

  1. Cliente envia requisição multipart/form-data (Postman, frontend, etc.)
  2. Spring converte para MultipartFile
  3. No Service:
    • Lê o conteúdo com InputStream
    • Processa (limpa linhas com regex)
    • Salva no diretório configurado

Slide 4: Código do Controller

@PostMapping("/arquivos")
@Operation(summary = "Upload de arquivos xml ou txt")
public ArquivoDto processaArquivo(@RequestPart("file") MultipartFile file) {
    return arquivosService.processaArquivo(file);

Slide 5: Processamento no Service

String conteudo = new BufferedReader(
    new InputStreamReader(file.getInputStream(), StandardCharsets.UTF_8))
    .lines()
    .map(linha -> linha.replaceAll("([^<]*?)([^<]*?)\\s*$", "$2"))
    .filter(linha -> !linha.isEmpty())
    .collect(Collectors.joining("\n"));

Remove prefixos indesejados e linhas vazias.

Slide 7: Melhores Práticas com MultipartFile

  • Configure limites em application.properties: 
    spring.servlet.multipart.max-file-size=10MB
spring.servlet.multipart.max-request-size=10MB
  • Validar tipo de arquivo e tamanho cedo
  • Prefirir streaming para arquivos grandes
  • Tratar exceções de IO melhor
  • Usar armazenamento em nuvem para produção AWS
  • Acrescentar timestamp ou contador ao nome para evitar exlusão manualmente

Obs: Possíveis melhorias apontadas para restringir e limitar o grande volume
